# 文档自动生成功能实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 集成 Doxygen 到 CMake 构建流程，从现有 Doxygen 风格注释自动生成 HTML API 文档。

**架构：** 使用 CMake 的 `find_package(Doxygen)` 检测 Doxygen，`configure_file` 生成带路径变量的 `Doxyfile`，通过 `add_custom_target` 将文档生成绑定到默认构建目标。

**技术栈：** CMake 3.10.0+，Doxygen（可选）

---

## 文件结构

| 文件 | 操作 | 说明 |
|------|------|------|
| `docs/Doxyfile.in` | 创建 | Doxygen 配置模板，含 CMake 变量占位符 |
| `CMakeLists.txt` | 修改 | 添加 Doxygen 检测与文档生成目标 |
| `AGENTS.md` | 修改 | 新增"注释风格"章节 |

### Task 1: 创建 Doxygen 配置模板

**Files:**
- Create: `docs/Doxyfile.in`

- [ ] **Step 1: 写入 Doxyfile.in**

创建 `docs/Doxyfile.in`，内容如下：

```ini
# Doxyfile for stringplus

PROJECT_NAME      = "stringplus"
PROJECT_BRIEF     = "小型 C 字符串辅助函数库"

INPUT             = @CMAKE_SOURCE_DIR@/src \
                    @CMAKE_SOURCE_DIR@/include \
                    @CMAKE_SOURCE_DIR@/README.md

OUTPUT_DIRECTORY  = @CMAKE_BINARY_DIR@/docs

GENERATE_HTML     = YES
HTML_OUTPUT       = html
GENERATE_LATEX    = NO

EXTRACT_ALL       = YES
EXTRACT_STATIC    = NO

USE_MDFILE_AS_MAINPAGE = @CMAKE_SOURCE_DIR@/README.md
```

- [ ] **Step 2: 验证文件创建**

Run: `cat docs/Doxyfile.in`
Expected: 显示上述内容，包含 `@CMAKE_SOURCE_DIR@` 和 `@CMAKE_BINARY_DIR@` 占位符

- [ ] **Step 3: Commit**

```bash
git add docs/Doxyfile.in
git commit -m "docs: 添加 Doxygen 配置模板"
```

### Task 2: 修改 CMakeLists.txt 集成文档生成

**Files:**
- Modify: `CMakeLists.txt`

- [ ] **Step 1: 读取当前 CMakeLists.txt**

Run: `cat CMakeLists.txt`
Expected: 确认现有内容，找到插入位置（文件末尾或 `install()` 之后）

- [ ] **Step 2: 追加文档生成逻辑**

在 `CMakeLists.txt` 末尾添加：

```cmake
# Documentation generation with Doxygen
find_package(Doxygen)
if(DOXYGEN_FOUND)
    configure_file(
        ${CMAKE_SOURCE_DIR}/docs/Doxyfile.in
        ${CMAKE_BINARY_DIR}/Doxyfile
        @ONLY
    )
    add_custom_target(docs ALL
        COMMAND ${DOXYGEN_EXECUTABLE} ${CMAKE_BINARY_DIR}/Doxyfile
        COMMENT "Generating API documentation with Doxygen"
        VERBATIM
    )
else()
    message(STATUS "Doxygen not found. Documentation will not be generated.")
endif()
```

- [ ] **Step 3: 验证 CMakeLists.txt**

Run: `cat CMakeLists.txt`
Expected: 原内容不变，末尾新增 `find_package(Doxygen)` 块

- [ ] **Step 4: 测试 CMake 配置（有 Doxygen 时）**

```bash
rm -rf build && mkdir build && cd build
cmake ..
```

Expected:
```
-- Found Doxygen: /usr/bin/doxygen (found version "...")
-- Configuring done
-- Generating done
```

- [ ] **Step 5: 测试构建并验证文档生成**

```bash
cd build && make
```

Expected: 构建输出中出现 `Generating API documentation with Doxygen`，完成后 `build/docs/html/index.html` 存在

Run: `ls build/docs/html/`
Expected: 包含 `index.html` 和多个 `.html` 文件

- [ ] **Step 6: Commit**

```bash

```

- [ ] **Step 2: 在"关键约定"后插入"注释风格"章节**

在 AGENTS.md 的"关键约定"章节末尾（`ccary_foreach()` 行之后）插入：

```markdown
## 注释风格

- 所有公共 API 使用 [Doxygen](https://www.doxygen.nl/) 风格注释，便于自动生成文档
- 格式要求：
  - 使用 `/** ... */` 块注释
  - 使用 `@brief` 描述函数功能
  - 使用 `@param` 描述参数（格式：`@param name : 说明`）
  - 使用 `@return` 描述返回值
- 示例：
  ```c
  /**
   * @brief index the first appearance of @ch in @str
   *
   * @param str : c-style string
   * @param ch  : char to been index
   * @return NOP if @str hasn't @ch, otherwise the index
   */
  size_t indexof(const char *str, int ch);
  ```
```

Expected: 包含新增的"注释风格"章节，示例代码格式正确
